# WNBA Edge Detector v1.0 Agent Interview

## Product Scope

WNBA Edge Detector v1.0 provides full-game matchup intelligence for WNBA games.

It should stay model-first until the historical lab proves the signal quality.

Recommended order:


## Initial WNBA Thresholds

These are test defaults, not final model claims.

```text
A side edge: model margin >= 7.0
B side edge: model margin >= 5.0
C side edge: model margin >= 3.0
Scoring environment threshold: +/- 4.0 points vs rolling league baseline
Default total baseline: 164.0
Recent margin weight: 0.20
```

Confidence should be gated by minimum prior team history:

```text
A requires both teams to have at least 5 prior games
B requires both teams to have at least 3 prior games
C can appear before both teams reach 3 prior games
```

The purpose of the gate is to avoid overstating early-season confidence.

## Historical Data Contract

The first WNBA backtest path expects one row per completed game:

```text
season
game_date
away_team
home_team
away_score
home_score
```

Optional:

```text
game_id
away_rest
home_rest
```

Rest can be calculated upstream or omitted. If rest is omitted, the v1.0 model should behave as a no-rest-context ablation.

## Rolling Team Strength

Starting formula:

```text
Team Strength =
0.50 * offense edge
+ 0.50 * defense edge
+ 0.42 * season margin
+ 0.20 * recent margin
```

Definitions:

```text
offense edge = team points for average - league points per team
defense edge = league points per team - team points allowed average
season margin = team average scoring margin
recent margin = average scoring margin over the last 5 games
```

The recent margin weight starts at 0.20 because WNBA recent form can matter, but v1.0 should avoid overreacting to a short sample.

Rest principle:

```text
Rest is tested, not assumed.
If rest improves backtest quality, keep it.
If rest adds noise, remove it from v1.0.
```

Implemented helper tables:

```text

```

These are generated by `wnba_agent.historical_summary_tables(results)` after the WNBA historical lab rows are built.

Run the historical report from a normalized CSV with:

```powershell
python wnba_agent.py --games-csv path\to\wnba_games.csv --season 2025 --compare-no-rest
```

To save report artifacts:

```powershell
python wnba_agent.py --games-csv path\to\wnba_games.csv --season 2025 --compare-no-rest --output-dir reports\wnba_2025
```

## Promotion Criteria

The v1.0 model is ready for Current Slate only if:

```text
Side signal accuracy beats all-game winner accuracy
A/B confidence tiers outperform C
High/Low scoring signals show useful directional accuracy
Road-edge performance does not collapse
Signal volume is large enough to be useful
Rest context is either validated or removed
```

If rest context fails to improve results, remove it from v1.0 rather than keeping it for narrative value.

## NBA Relationship

Do not assume the WNBA model can be reused for NBA without separate validation.

Reusable:

```text
Data contract
Backtest engine shape
Agent wrapper shape
UI card contract
Validation report structure
```

Not automatically reusable:

```text
Thresholds
Home-court value
Rest effect
Recent-form weight
Scoring baseline
Signal-volume expectations
```

The WNBA model should be kicked first, then the NBA model should get separate parameters and validation.
